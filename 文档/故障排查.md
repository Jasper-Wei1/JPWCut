# 故障排查

## Windows 安装或 Studio 启动失败

Windows 10/11 x64 用户从仓库根目录双击 `Windows/install.cmd`，不要在 OneDrive、桌面、含中文用户名或过深目录中运行。推荐解压到 `D:\JPWClips`。安装脚本会检查 Node.js、PowerShell、Remotion 和本地 Whisper。

Windows ARM 设备暂不在支持范围内，因为本项目的本地 Whisper 安装包使用 x64 二进制。

## `npm run setup` 无法编译 Whisper.cpp

本节只适用于 macOS 或 Linux。

先安装 Apple 命令行开发工具，再重试：

```bash
xcode-select --install
npm run setup
```

## 模型下载中断

重新运行同一个安装命令。完整模型按精确文件大小校验；不完整文件会移入 `工作区/待删除/`，不会被误用。

## 转录提示找不到输入文件

从仓库根目录运行命令，并给中文或带空格的路径加双引号。直播原片应位于 `输入/媒体素材/直播录像/`。

## Windows `small` 模型出现乱码或异常时间码

Windows 上的 Whisper.cpp 1.5.5 二进制在启用 DTW token 级时间戳时，可能产出损坏的中文 token、异常长字幕或超出原片时长的时间码；这不是 `small` 模型本身失效。项目在 Windows 自动关闭 token 级时间戳，并按 128 个 token 分段。macOS 和 Linux 保持 token 级时间戳。

修改 Whisper、模型或平台相关参数后，先用一小段代表性录音验证，再处理整场直播：检查标准化逐字稿中没有乱码替换字符、没有超长字幕，且所有时间码都在原片时长范围内。通过后再运行完整转录，避免长时间任务失败后才发现问题。

## Studio 找不到直播源

先完成 `npm run clips:review -- prepare ...`。该步骤会将原片链接或复制到 Remotion 的本地生成目录，原片本身不会修改。
