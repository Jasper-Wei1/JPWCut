# 直播切片 V1 需求

状态：V1 已实现并通过工作流验证。

## 目标

输入一段已保存的直播录像，Agent 使用本地 Whisper.cpp 和有证据的内容分析
提出高光候选。用户在 Studio 确认源区间后，工作流为每个已批准区间生成独立
的 9:16 短视频。

这是本仓库唯一的技术工作流 `clip-extraction`，直接从一段直播原片中筛选内容。

## 已确认的 V1 决策

- 默认输出抖音和小红书使用的 9:16 竖屏视频。
- 候选评分必须覆盖完整原片时间轴，不得在找到若干高分片段后提前停止。
- 只保留总分严格大于 85 的候选，排序去重后数量不设上限。
- 每条输出只能来自原直播中的一个连续区间。
- V1 不拼接距离较远的句子、不调整语序，也不合并多场直播。
- 原直播录像不可修改，并且始终是唯一主音频来源。
- 候选选择和最终视觉成片都需要用户明确确认。
- 最终内容标题需要第三个独立的文本确认门，只用于成片文件名。

## 输入

必需：

- `输入/媒体素材/直播录像/` 下的一段已保存直播录像。

可选：

- 目标平台。
- 期望切片数量。
- 最短和最长时长。
- 优先或排除的话题。
- 说话人姓名和需要重点检查的专有名词。

默认用 90 秒窗口、60 秒步长对完整原片进行重叠覆盖，每个区间都必须评分。
去重后的高分候选可以为 0 条或任意多条，不设五条上限。每条批准源区间仍必须
是 30 至 90 秒，锁定派生母版时长与批准源区间一致。输出固定为
1080x1920、30fps。

## 高光定义

“高光”不能只靠主观感觉，每个候选必须根据逐字稿证据评分：

| 维度 | 权重 | 通过证据 |
| --- | ---: | --- |
| 独立完整性 | 30 | 片段包含必要背景、核心观点和收束。 |
| 信息价值 | 20 | 包含方法、答案、案例、结果或明确判断。 |
| 开头强度 | 20 | 前 3 至 5 秒出现问题、冲突、结果或强判断。 |
| 表达强度 | 10 | 表达集中，并出现有意义的情绪或语气变化。 |
| 可剪辑性 | 10 | 开头和结尾位于安全句子、词语或静音边界。 |
| 平台适配 | 10 | 内容适合设定时长和竖屏呈现。 |

保留线是总分严格大于 85。85 分本身不进入候选。不得为了凑数降低阈值、
截断为前若干条或补造上下文。

## 全时间轴评分审计

- 脚本先生成 90 秒窗口、60 秒步长的重叠区间，第一个区间从 0 开始，
  最后一个区间必须到达原片结尾。
- `coveragePercent` 必须为 100，`gaps` 必须为空，否则不得生成候选报告。
- 每个区间必须保存主题键、工作标题、核心观点、六维分数、原话、证据、
  风险和淘汰理由。
- 总分不高于 85 的区间必须写具体内容淘汰原因，不得只写“分数不够”。
- 评分审计保留全部区间，包括入选、低分淘汰、时长不合格和去重淘汰。
- 先按总分降序，再按开头强度降序，最后按原片起点排序。
- `topicKey` 相同或两个区间占较短区间的重叠比例达到 50% 时，保留更高分项。
  被去重的区间保留 `duplicateOf` 和具体原因。

## 候选数据

每个候选必须包含：

- 候选 ID、原片开始和结束时间、预计时长。
- 建议标题和一句话核心观点。
- 对应区间的原话逐字稿。
- 分项得分和具体入选证据。
- 上下文、隐私、敏感表达和歧义警告。
- 附近上下文及建议边界调整。
- 与其他候选的重叠比例。

排序去重后仍存在的少量重叠会写入 `overlapWith`，供用户审核。

## 最终标题数据

标题确认稿必须让用户能直接阅读，每条至少包含切片 ID、推荐标题、
`dbs-xhs-title` 公式编号和一句内容依据。用户可以逐条确认，也可以明确
说“全部确认”。

确认后持久化为：

```json
{
  "schemaVersion": 1,
  "workflow": "livestream-content-title-review",
  "id": "<name>",
  "status": "approved",
  "approvedAt": "<ISO timestamp>",
  "titles": [
    {
      "clipId": "clip-001",
      "contentTitle": "<20 字以内的已确认标题>",
      "formulaSkill": "dbs-xhs-title",
      "formulaId": 1
    }
  ]
}
```

标题数据必须覆盖当前批次的全部已批准切片，不得多条切片重名，不得包含
路径分隔符或其他不可移植文件名字符。

## 确认语义和范围

| 用户说法 | 当前上下文 | 只批准什么 |
| --- | --- | --- |
| 全部保留 | 候选审核稿 | 选中当前全部候选。 |
| 全部切点确认 | 切点 Studio 审核 | 批准当前全部连续原片区间。 |
| 确认通过 | 最终视觉审核 | 批准 9:16 裁切、字幕和画面处理。 |
| 全部确认 | 标题确认稿 | 批准当前列出的全部最终内容标题。 |

任何确认只对当前已展示的审核层级生效，不得跨越后续尚未展示的确认门。
用户后续缩小或纠正范围时，以最新明确说法为准。

## 固定流程

```text
不可修改的直播原片
-> 本地 Whisper.cpp 逐字稿和可复用缓存
-> 全时间轴重叠窗口审计
-> 每个区间完成六维评分和淘汰理由
-> 保留所有评分记录
-> 筛选总分严格大于 85 的区间
-> 降序排列并按主题和时间重叠去重
-> 数量不设上限的候选报告
-> 用户选择候选
-> Studio 检查源区间、边界和完整播放结果
-> 用户批准区间
-> 每个批准区间生成独立派生母版并锁定时间轴
-> 将原画面居中裁切为 9:16 并叠加字幕
-> Studio 检查最终视觉视频
-> 用户确认最终效果
-> 根据每条实际内容生成最终内容标题
-> 用户确认全部标题
-> 渲染最终成片
```

第一次确认决定允许提取哪些连续源区间；第二次确认决定 9:16 居中裁切、字幕和
视觉处理是否可以最终渲染。沉默或未回复不代表同意。

候选标题、候选编号、分数、审核状态和原片时间码只用于候选与切点审核，
不得进入最终视频数据或成片画面。

标题确认是独立的文本确认门，不需要在 Studio 中再做一次视觉审核。
最终内容标题必须从该切片实际内容中提炼，默认不超过 20 个字符，并使用
项目内 `skills/dbs-xhs-title/SKILL.md` 生成可追溯的公式编号。不得为了套公式
虚构切片中没有的事实、数字或结果。

最终内容标题不是画面元素，不写入 Remotion 成片数据，只用于
`<contentTitle>.mp4` 文件名。若成片已经渲染，后续只修改标题时必须无损重命名
并比对哈希，不得重新渲染或改变画面。

## 时间轴和音频边界

- 切片开头或结尾可以移动到附近静音或词语边界。
- V1 不得拼接不连续区间，也不得改变原话语序、语速、措辞或含义。
- 逐字稿审校只能修正识别错误和专有名词，不得改写说话内容。
- 切点确认后，才生成派生切片母版并锁定时间轴；模板阶段不得再次缩短、移动
  或重排内容。
- 原直播声音是唯一主音频，不包含 AI 换声和未经确认的音乐。

## 规划的数据和输出

- 全时间轴评分审计：`工作区/数据/草稿/<name>-scoring-audit.json`
- Agent 评分决定：`工作区/数据/草稿/<name>-scoring-decisions.json`
- 已评分完整审计：`工作区/数据/草稿/<name>-scored-audit.json`
- 候选草稿：`工作区/数据/草稿/<name>-clip-candidates.json`
- 可直接阅读的完整评分审核稿：`工作区/数据/草稿/<name>-全时间轴评分审核稿.md`
- Studio 审核数据：`引擎/remotion/public/workflow/clip-review-current.json`
- 已批准方案：`工作区/数据/已确认/<name>-approved-clips.json`
- 已确认标题：`工作区/数据/已确认/<name>-content-titles.json`
- 派生母版：`工作区/派生媒体/直播切片/<name>/<clip-id>/master.mp4`
- 预览：`输出/预览/<name>-<clip-id>-preview.mp4`
- 成片：`输出/最终成片/<contentTitle>.mp4`
- 审核记录：`工作区/质检复盘/`

## 必需实现面

- 项目 Skill：`skills/extract-livestream-clips/SKILL.md`
- 确定性的分析、审核和应用命令。
- 独立的候选数据和批准方案结构。
- 可拖动的 Studio 候选审核 Composition。
- 技术 ID 为 `livestream-clip-916` 的模板、说明、示例数据和画廊图片。
- 覆盖评分、边界、确认门禁、原片不可变、输出媒体和模板登记的测试。
- 只有工作流真实可运行后，才能更新 `AGENTS.md` 路由和 README 新手说明。

## 验收标准

- 运行时音频不会离开本机进行转录。
- 未变化的直播录像会复用匹配逐字稿缓存。
- 每条渲染切片只映射一个连续源区间。
- 锁定派生母版时长等于批准源区间时长。
- 待确认或已拒绝候选不得生成输出。
- 评分审计覆盖率为 100%，且每个区间都已评分并保存去向。
- 总分不高于 85 的区间和去重淘汰区间都保留具体原因。
- 严格大于 85 且去重后的所有区间均进入候选，没有数量上限。
- 所有应用区间都经过明确 Studio 确认并具备安全边界。
- 每条成片都有一个明确确认、内容真实、不重名的最终内容标题。
- 每条成片都有有效的 1080x1920 视频流、源录像音频、批准时长、可读字幕，
  且没有错误裁切或元素重叠。
- 实现后 `npm run check` 必须通过。

## 非目标

- 实时获取或处理直播流。
- 自动发布。
- 多机位切换或说话人分离。
- 跨录像拼接或不连续句子组装。
- 自动删除源录像、候选、日志或最终成片。
- 未经明确授权添加音乐或第三方素材。
