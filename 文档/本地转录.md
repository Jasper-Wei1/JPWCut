# 本地语音转录

直播原片经本地 Whisper.cpp 转为带时间码的逐字稿：

```text
直播录像 -> 16 kHz 单声道 WAV -> Whisper.cpp -> 标准化逐字稿 JSON
```

音频不会上传到云端语音服务。直播没有正式口播稿；原声和时间码逐字稿是内容真源。只能依据证据修正明确的错字或专名，听不清的内容必须记录疑点，不得为流畅改写原话。

默认模型是 `small`，约 488 MB。草稿结果放在 `工作区/数据/草稿/`，已确认的审校逐字稿放在 `工作区/数据/已确认/`。

## Windows 转录兼容性与质量门槛

Whisper.cpp 1.5.5 的 Windows `main.exe` 在中文 DTW token 时间码模式下可能输出 `U+FFFD` 乱码和跨整片的异常 token 时间码。因此 Windows 禁用 DTW，改为句段字幕，每段最多 128 tokens；macOS 和 Linux 保持 token 时间码。

任何 Windows Whisper 二进制、模型或转录参数变更，都必须先从同一原片抽取开头、中段和结尾各 30 秒进行转录。对 `toCaptions -> normalizeWhisperCaptions` 的实际输出逐项检查：

- 不得含有 `U+FFFD` 字符。
- 不得有 `endMs <= startMs` 的字幕。
- 不得有持续时间超过 30 秒的单条字幕。

三个样本全部通过后，才允许执行整片转录。整片结果也必须通过同一检查，未通过不得进入全时间轴评分、候选审核或 Studio。

只检查三段样本而不生成或覆盖整片逐字稿时，运行：

```bash
npm run transcribe -- --input "输入/媒体素材/直播录像/<video>.mp4" --model small --preflight-only
```
