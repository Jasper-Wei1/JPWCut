# 本地直播精彩切片工作流

这是一个只处理已保存直播录像的本地仓库。它使用本地 Whisper.cpp 完成带时间码的逐字稿，覆盖完整原片时间轴筛选候选，并用 Remotion 在 Studio 中完成两次视觉确认后渲染 9:16 成片。

原始声音不会上传到云端语音服务；原直播录像始终不可修改。

## 你只需要认识两个位置

```text
输入/媒体素材/直播录像/     放一段已保存的直播原片
输出/最终成片/               取已确认的竖屏切片
```

## 安装

最低要求：macOS、Node.js 20+、Git、Xcode Command Line Tools。

```bash
npm run setup
npm run doctor
```

首次安装会下载依赖、编译本地 Whisper.cpp，并下载默认的 `small` 中文模型，约 500 MB。

## 制作一场直播的精彩切片

把一段录像放进 `输入/媒体素材/直播录像/`，然后对 Agent 说：

```text
请读取 skills/extract-livestream-clips/SKILL.md，从
输入/媒体素材/直播录像/<视频文件>.mp4 筛选精彩切片。
先交付候选审核稿；切点、冷开场和最终竖屏视觉都必须在 Studio 确认，不要直接渲染成片。
```

固定流程：

```text
不可修改的直播原片
-> 本地 Whisper.cpp 逐字稿
-> 覆盖 100% 时间轴的六维评分
-> 候选审核
-> Studio 确认连续源区间和片内冷开场
-> 独立派生母版并锁定时长
-> Studio 确认 9:16 裁切和字幕
-> 最终内容标题确认
-> 渲染成片
```

每条成片只来自一段已批准的连续原片区间。唯一例外是片内冷开场：可从同一区间复制 2 至 8 秒原画面、原声音和字幕到开头，正文仍从原起点完整播放。最终标题只用于文件名，不会写入画面。

## 常用命令

```bash
# 环境检查
npm run doctor

# 本地转录一场直播
npm run transcribe -- \
  --input "输入/媒体素材/直播录像/<视频文件>.mp4" \
  --name "<批次名称>"

# 打开 Studio 审核
npm run studio

# 重新生成直播模板画廊
npm run gallery

# 运行全部验证
npm run check
```

完整边界与验收标准见 [直播切片需求](文档/直播切片需求.md)。
